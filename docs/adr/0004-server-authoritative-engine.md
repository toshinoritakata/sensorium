# コアエンジンはサーバ側を権威（authoritative）として動かす

フィジビリティのコア計算（Spec → 11次元 Condition＋Setup 候補、What-if 再計算）はサーバ側で実行し、そこを正とする。クライアントは入力・操作・結果表示を担う。エンジンは引き続き純粋パッケージ（ADR-0003）で、サーバがそれを import して動かす。

ADR-0001 はエンジンを決定論に保つことで「オフライン・即時・再現可能・課金なし」を挙げていたが、本決定はそのうち**オフライン／ブラウザ即時**を意図的に手放す。代わりに得るもの: ①大きく育つ機材DB（取込：ADR-0002）とエンジンを同居させ、全カタログをクライアントへ送らずに照合できる ②判定ロジック（IP）を秘匿できる ③重い計算をしっかり回せる ④DB・取込・Claude proxy と一体で構成が単純。決定論・再現性・課金なし（What-if が LLM 非依存）という ADR-0001 の核は保たれる。

帰結: What-if はクライアント→サーバ往復になる。スライダー等はデバウンス＋高速エンドポイントで実用的な体感に収める。将来 What-if の即時性が要件化したら、同じ純粋パッケージをクライアントにも積んで「サーバ権威＋クライアント即時プレビュー」のハイブリッドへ拡張できる（パッケージが純粋なので二重実装にならない）。この配置決定はフレームワーク選定をサーバ中心（フルスタック）へ強く傾ける。
